# Summarize Conversation

Generate a focused, executive-style summary of the current conversation and save it to `~/Downloads/`.

## Instructions

1. Read `$ARGUMENTS` — this is the **focus directive**. It tells you exactly what the reader cares about. Only include information relevant to this directive. If no arguments given, ask the user what they want the summary to focus on.
2. Derive a filename slug from the focus (e.g., `sse-reconnect-events`, `duplicate-task-root-cause`)
3. Write the summary following the format below
4. Save to `~/Downloads/YYYY-MM-DD-{slug}.md`
5. Tell the user the full path

## Output Format

```markdown
# {Title derived from focus directive}

**Date:** {YYYY-MM-DD}
**Context:** {One line — what conversation/system/ID was being investigated}

## TL;DR

{2-3 sentences. The answer to the focus directive. A busy executive should be able to stop reading here.}

## Details

{Focused narrative covering ONLY what the arguments asked for. Use tables for chronological/comparative data. Use code blocks for log lines or trace IDs. Keep it tight — every sentence must earn its place.}

## How It Works

{Include ONLY if the focus directive asks about mechanism/behavior. Explain the relevant system behavior concisely — reference code paths with `file:line` format.}

---
_Generated by Claude Code on {YYYY-MM-DD}_
```

## Constraints

- **Under 80 lines** — this is an executive summary, not a report
- Only include what the focus directive asks for — ruthlessly cut everything else
- No boilerplate sections — omit any section that has nothing to contribute
- No "Next Steps" unless the user explicitly asks for recommendations
- No API keys, auth tokens, or PII
- Use facts from the conversation only, never speculate

ARGUMENTS: $ARGUMENTS
